## 文档边界说明

本文档描述的社区层功能分为两类，归属不同系统：

**外骨骼通用社区框架（本文档描述）**：
- 论坛讨论系统（发帖/回复/点赞/@用户）
- 用户关注/粉丝系统
- 动态流（用户活动时间线）
- 评论系统（Markdown编辑/嵌套回复）

> 这些是任何会员系统都有的通用能力，由外骨骼系统硬编码实现。

**核心透传功能（不在本文档描述，由核心通过配置接口驱动）**：
- Fork 按钮（核心配置"显示Fork按钮，调用/core/fork"）
- MR 按钮（核心配置"显示MR按钮，调用/core/mr"）
- 仿真预览（核心提供iframe/组件，外骨骼嵌入）
- 研究进展（核心提供数据，外骨骼渲染）
- 排行榜（核心提供排序数据，外骨骼通用列表组件渲染）

> 外骨骼只负责"渲染按钮/组件 + 转发请求到核心API"，不实现 Fork/MR/仿真的业务逻辑。
> 核心在 `/core/fork`、`/core/mr` 等 API 内部处理所有业务逻辑。
> 核心配置接口：`GET /api/v1/core/page-config?type=project-public&project_id={id}`

- 星标按钮：点击 toggle，数字实时更新

**排序算法**（综合热度分）：
```
score = (stars * 3) + (forks * 5) + (comments * 2) + (recent_ticks * 0.001)
        + (expert_badge_boost * 10) + (time_decay_factor)

time_decay_factor = max(0.3, 1 - (days_since_last_activity / 30))
```

### 7.20 个人空间与个人项目运行（Personal Workspace）

> **归属**：核心透传。Fork、MR、仿真运行、项目对比均为核心特有功能。
> 外骨骼只负责渲染页面容器，所有业务逻辑由核心 `/core/fork`、`/core/mr` 等 API 处理。
> 外骨骼通过核心配置接口接收页面区块配置（按钮显示/隐藏、调用目标API）。

**"打开"按钮的行为**：

| 用户类型 | 行为 |
|----------|------|
| **访客（未登录）** | 弹出登录/注册模态框 |
| **已登录，首次打开** | 自动 Fork 项目到个人空间 → 进入个人工作台的只读预览模式 |
| **已登录，已有 Fork** | 直接进入自己的 Fork 版本 |
| **项目所有者** | 直接进入原始项目的编辑模式 |

### 7.21 专家认证与排行榜 ★新增

> **归属**：混合。专家认证体系（徽章、条件、权益）由外骨骼定义和管理；排行榜数据和声誉分数由核心提供。
> 外骨骼提供通用列表组件渲染排行榜；核心通过 `GET /api/v1/core/leaderboard` 提供排序数据。
> 贡献徽章系统中涉及核心指标（代谢健康分、觅食产出率）的部分由核心计算，外骨骼只负责展示。

**页面**：`/#/experts` 和领域广场侧边栏

**声誉体系维度**：

| 维度 | 说明 | 权重 |
|------|------|------|
| **项目质量分** | 公开项目的星标、Fork、被合并次数 | 30% |
| **验证贡献分** | 验证通过的假设数量、引用次数 | 25% |
| **知识贡献分** | 贡献到知识条目市场的条目数、被复现次数 | 25% |
| **代谢健康分** ★ | 项目的代谢健康指标（觅食产出率/过滤通过率/同化成功率） | 20% |

**代谢健康贡献维度说明**：
- 不仅看项目的" popularity "（星标/Fork），更看项目的"研究质量"
- 一个项目如果代谢健康指标优秀（高觅食产出率、合理的过滤通过率、高同化成功率），说明其研究过程严谨、自动化程度高、成果可信
- 代谢健康分由系统根据项目运行期间的代谢指标自动计算，不可人为刷分
