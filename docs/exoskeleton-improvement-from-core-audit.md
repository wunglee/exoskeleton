### 1.1 用户与订阅体系（来自 `01-users-concepts.md`）

#### 1.1.1 研究积分（RP）系统

**完整定义**：
- 研究积分（RP）是平台内货币，**唯一来源**是个人项目的三项表现：
  - 公众评价（星标、高质量评论）
  - 关注度（浏览量、被 Fork、被打开运行）
  - 对其他项目的实际贡献（合并请求被接受）
- 发布项目本身不产生积分，积分来自项目被社区认可的程度
- 合并请求被接受是最高价值来源（原作者认可你的改进并吸收）
- 积分可用于抵扣会员费（最高 50%）、兑换 Tick/Token/存储增量包
- 积分每日微衰减 0.1%，激励持续产出
- 个人荣誉级别（探索者→学徒→研究者→学者→首席科学家）仅由上述三维度产生的累计积分决定，终身成就制，兑换消费不影响级别
- 荣誉级别显示于用户头像旁（铜/银/金/钻石边框），解锁平台特权

**积分来源明细**：

| 维度 | 指标 | 积分 |
|------|------|------|
| 公众评价 | 星标 | +10 RP/星（每个用户仅计一次） |
| | 高质量评论 | +30 RP/条（≥100字且被 3+ 点赞） |
| 关注度 | 浏览量 | +0.01 RP/次（UV去重） |
| | 被 Fork | +50 RP/Fork |
| | 被打开运行 | +20 RP/次 |
| 实际贡献 | 合并请求被接受 | +250~1000 RP（按影响权重） |
| | 发布首个公开项目 | +200 RP（一次性） |
| 衰减 | 每日被动消耗 | -余额 × 0.1% |
| 上限 | 每日获得上限 | 2000 RP |

### 1.2 注册与认证流程（来自 `03-workflows.md`）

**完整流程**：

```
访客点击[注册]
    ↓
选择注册方式
    ├── 邮箱注册 → 填写邮箱/密码 → 验证邮件 → 完善资料
    ├── OAuth (Google/GitHub) → 授权 → 绑定邮箱 → 完善资料
    └── 手机号注册（中国区）→ 短信验证码 → 设置密码
    ↓
选择角色标签（可多选）：首席研究员 / 领域建模师 / 数据分析师 / 策略验证员 / 学生
    ↓
进入引导式创建首个项目
```

> **迁移目标**：`01-users-auth.md` 第2.3节（手机号注册）

### 1.3 核心系统向外骨骼提供的配置接口

核心应定义以下配置数据，供外骨骼读取：

```
GET /api/v1/core/page-config?type=project-public&project_id={id}
→ 返回页面区块配置（按钮/Tab/列表等）

GET /api/v1/core/permissions?user_id={uid}
→ 返回用户功能开关（can_fork/can_mr/can_simulate等）

GET /api/v1/core/leaderboard?domain={id}
→ 返回排行榜数据（排名/分数/徽章URL）

POST /api/v1/core/projects/{id}/fork
→ 核心处理Fork业务逻辑

POST /api/v1/core/projects/{id}/merge-requests
→ 核心处理MR业务逻辑
```

外骨骼只调用接口、渲染结果，不解释业务含义。

- 论坛讨论系统（发帖/回复/点赞/@用户）

- 仿真预览（核心提供iframe/组件，外骨骼嵌入）

- 排行榜（核心提供排序数据，外骨骼通用列表组件渲染）

**重构后的 `03-community.md` 应只描述通用社区框架**：
- 论坛/评论/点赞/关注的具体实现
- 用户动态流的数据模型
- 社区页面的通用布局系统

核心透传功能不在外骨骼文档中描述，由核心通过配置接口驱动。

## 三、功能归属边界澄清

| 概念 | 归属 | 说明 |
|------|------|------|
| **Branch** | 核心 | 研究假设分支（draft/baseline/creative/ablation），含Body I/II/III验证 |
| **Fork** | 核心（透传） | 核心特有功能，外骨骼通过配置渲染按钮，调用核心API |
| **MR** | 核心（透传） | 核心特有功能，外骨骼通过配置渲染按钮，调用核心API |
| **论坛/评论/点赞** | 外骨骼 | 通用社区框架，任何会员系统都有 |
| **仿真预览** | 核心（透传） | 核心提供预览组件，外骨骼嵌入渲染 |
| **排行榜** | 混合 | 外骨骼提供通用列表组件，核心提供排序数据 |

**关键澄清**：外骨骼通用社区框架**没有 Fork/MR 功能**。Fork/MR 是核心特有的项目协作功能：
- 外骨骼通用社区框架**不包含** Fork/MR 的实现代码
- Fork/MR 由核心通过配置透传："这里显示一个按钮，文字Fork，调用/core/fork"
- 外骨骼只知道"渲染按钮+转发请求"，不知道这是Fork
- 核心在 `/core/fork` API 内部处理所有业务逻辑
